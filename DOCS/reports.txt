Строение .torrent-файла довольно простое, он представляет из себя словарь в формате bencode. 
В данном словаре нас интересует только пара с ключом info (блок описания файлов). 
Этот тоже является словарем и содержит в себе информацию об имени файлов, их размере.
Кроме того, торрент хеширует файлы не целиком, а по кускам определенной длины, которая зависит от размера этих файлов. 
Информация о размере этого куска также содержится в словаре info.
После того, как я разобрался с устройством .torrent-файлов, передо мной встал вопрос парсинга всего этого чуда. 
По данному вопросу существуют несколько под это дело:
- библиотека BencodeLibrary
Что же есть в нише BitTorrent-библиотек для Java:
- Azureus, ныне Vuze (торрент-клиент)
- Snark
- jBitTorrent API
Я заострил свое внимание на мавен-проекте "com.fasterxml.jackson". 
Она позволяет делать разбор torrent-файлов и собирать заголовочный файл в b-encode формате. 
Сравнивая ее с другими библиотеками хочу отметить, здесь богатый функционал для решения подобных задач.
Для удобства пользуюсь утилилитой-редактором заголовочных torrent-файлов (удобна для визуализации результата контекста файла...).
(Для хранения информации из прочитанного файла использую класс-Torrent...)
